# 모바일 웹 제한 및 QA 체크리스트

## 브라우저 / OS 제약

- **HTTPS**: Geolocation과 카메라는 보안 컨텍스트(HTTPS 또는 `localhost`)에서만 안정적으로 동작합니다. Firebase Hosting 배포 URL을 사용하세요.
- **권한 UX**: 위치·카메라는 시스템 팝업으로 허용해야 합니다. 거절 시 기능이 멈추므로 안내 문구를 확인합니다.
- **iOS Safari**: 사용자 제스처(본 앱의 **Go** 버튼) 이후 `getUserMedia` 호출이 권장됩니다. 인카운터 진입 시에도 카메라를 엽니다.
- **후면 카메라**: `facingMode: environment`는 기기마다 무시되거나 전면으로 열릴 수 있습니다.
- **GPS 오차**: 실내·고층에서는 오차가 커질 수 있습니다. `getImage.html`의 `PROXIMITY_M`을 현장에 맞게 조정하세요.
- **백그라운드**: 웹은 앱처럼 백그라운드에서 위치를 계속 쓰기 어렵습니다. 포그라운드에서 플레이한다고 가정합니다.
- **익명 계정**: 브라우저 데이터 삭제 시 UID가 바뀌어 수집 목록이 비어 보일 수 있습니다.

## AR에 대해

- 본 프로젝트는 **카메라 풀스크린 + 화면 고정 오버레이**입니다. 포켓몬 고 앱과 동일한 월드 고정 AR은 웹만으로는 기기 편차가 큽니다.

## 실기 QA 체크리스트

다음은 배포 URL(HTTPS)에서 **실제 단말**로 확인할 항목입니다.

### 공통

- [ ] `dataupload.html` / `getImage.html` 모두 익명 로그인 성공(상단/스플래시 오류 없음).
- [ ] 위치 권한 허용 후 지도 중심이 내 위치로 이동한다.

### Android (Chrome 권장)

- [ ] 업로드: 이미지 선택 → 등록 → 완료 메시지.
- [ ] 플레이: Go → 지도 → 스폰 근접 시 카메라 + 오버레이.
- [ ] 탭 후 마커 소멸, 수집 목록에 항목 표시.

### iOS (Safari)

- [ ] Go 후 지도·위치 동작.
- [ ] 근접 시 카메라 권한 및 후면(또는 대체) 영상.
- [ ] 수집 후 목록·재등장 없음 동작.

### Firestore / Storage

- [ ] 콘솔에서 `spawns` 문서 필드가 규칙과 일치하는지.
- [ ] `caught` 경로가 `users/{uid}/caught/{spawnId}` 인지.
- [ ] Storage 객체가 `uploads/{uid}/…` 인지.

## 문제 발생 시

- 규칙 오류: Firebase 콘솔 → Firestore/Storage → Rules 시뮬레이터로 쓰기 요청 검증.
- 카메라만 실패: 다른 브라우저(Chrome iOS) 시도, iOS 설정에서 Safari 카메라 권한 확인.
